About

I'm a software engineer with almost 8 years of experience in web development, with a particular passion for backend and infrastructures. In my career I have designed, developed and coordinated almost 100 projects, from the simplest showcase sites to complex commercial and management systems for large companies.
I'm a maniac of detail, performance and best practices, always looking for new challenges to face, solutions to find and new technologies to experiment.

My passion for Linux-based systems was born when I was in seventh grade and my father gave me his old Compaq laptop, on which I installed Ubuntu 8: since then I have never stopped fiddling with Linux systems (I use Arch btw).
Still today I manage and administer, both for customers and for personal use, various servers with different distros depending on the needs.

I started programming in C as a self-taught in high school and, thanks to university, I learned new languages and became passionate about software engineering, code quality and database systems.
Thanks to the CyberChallenge.IT project, I discovered the world of cybersecurity and started playing CTF competitions. I am currently specializing in Cybersecurity at Politecnico di Milano.

Do you have a project in mind that you would like to realize? Contact me and let's talk about it together!

Skills

Backend

I am very familiar with the PHP language, which I have used to develop countless projects. The frameworks I've worked with most frequently are Symfony and Laravel.
I know the NodeJS ecosystem well, I have used Express for several projects and lately I have also used Deno for some microservices.
I also used Java mainly for API development, especially with the Spring framework.

The DBMS I've worked with the most are PostgreSQL and MySQL, but on more than one occasion I've also used SQLite for small projects and embedded systems.
I happened to use Redis as a persistence system for some real-time collaborative applications.
Lately I'm experimenting with document-oriented solutions like MongoDB and serverless technologies like the Firebase suite.

I developed a dozen Telegram bots.

I work daily with Python, an indispensable language for CTF but also used for some projects related to AI, in particular with the Keras framework.

Frontend

HTML, CSS and JavaScript cannot be missing from the knowledge of a web developer. I like to pay close attention to the semantically correct use of HTML elements and A11y standards, not only for SEO reasons but above all for a more accessible web for those who browse with alternative technologies such as screen readers.
I firmly maintain that in today's web there is too much JavaScript (on this page there is not even a line: go for it!).

For some highly interactive projects, I used SvelteKit and SolidJS.
